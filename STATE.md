# STATE — Where the Project Is Right Now
Last updated: 2026-07-20

PrintShelf is a 3D-print tracker (log prints, filaments, printers; public maker profiles; affiliate-monetized filament Buy-links) — sibling project to SS Book Tracker, built solo by Cam. FastAPI + SQLAlchemy/Alembic backend, Jinja2/HTMX-style server-rendered dashboard (no JS framework), PostgreSQL on Railway (SQLite locally), R2-backed uploads, freemium with a Stripe Pro tier (free caps: 50 prints / 10 filaments / 10 collections, enforced centrally in `backend/limits.py`). Live at printshelf.app (staging at staging.printshelf.app, auto-deploy from the `staging`/`main` branches on Railway). Currently mid-audit: session 33's full-screen audit (6 fixes) shipped to staging (`470b4f7`) and has **not yet been Cam-QA'd/merged to main** — the tip of `main` is one small fix ahead of that (`6e1ebad`, 2 days ago), so staging and main have diverged pending Cam's go-ahead.

## Working / Shipping
- **Core tracking loop**: printers, filaments, prints (with photos, slicer settings, cost tracking, ratings, categories), public per-user profiles at `/@username`, Explore/Feed/Search with facets (material/brand/printer) and time-decayed "Trending" + "Most liked" sorts.
- **Auto-import pipeline** (the main growth lever this year): URL import from Printables/Thingiverse/Cults3D/MakerWorld/MyMiniFactory, a slicer post-processing script (`tools/printshelf_postprocess.py` → `POST /api/prints/ingest`) for Bambu/Orca/Prusa, and a keyless mobile share flow (`GET /share?url=` via an iOS Shortcut) — all deduped by canonical model URL.
- **Filament capture flow** (2026-07-12, `34919d1`, live on prod): 3-tier picker on the print form — your filaments → community-sourced suggestions (cross-user distinct combos, no new table) → inline quick-add — plus a dismissible post-save nudge on `/dashboard/prints`. Built specifically because filament logging was the funnel's real bottleneck (~3 of 71 users were logging filaments, starving the affiliate Buy-link revenue this app is monetized on).
- **"Preview in Real Filament"** — client-side WebGL studio (`/dashboard/preview` + public no-login `/preview`) rendering a sample model or an ephemeral (never-persisted) uploaded STL in a real filament's color/finish, wired to affiliate Buy-links. Fully launched and live (`FILAMENT_PREVIEW_ENABLED` set on prod).
- **Growth/retention infra**: Google OAuth login, PWA install (manifest + service worker + offline page, confirmed installable on iPhone), Collections (named print groups), likes/views engagement loop, user data export (JSON/CSV), affiliate disclosure footer, Chrome extension (separate repo) for capturing filament purchases from Bambu/Polymaker/SUNLU/FlashForge/etc. product pages.
- **Latest shipped fix** (`6e1ebad`, 2 days ago): print detail's "View on X" link now uses a shared `platform_display_name()` helper (`backend/import_service.py`) recognizing 7 platforms (was 4 hardcoded) plus a domain-name fallback for anything else; also added the missing `myminifactory` value to the `SourcePlatform` enum (API/docs had drifted — `/developers` documented it but the enum 400'd on it).
- **QA discipline**: an 85-check automated suite (`backend/scripts/qa.py`) plus a Playwright visual-audit script (`backend/scripts/visual_audit.py`) run before every merge to main; CLAUDE.md's "Customer-Facing QA Gate" requires staging deploy → QA script → explicit Cam go-ahead before merging.
