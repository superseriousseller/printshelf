🔌 Connect your slicer

Auto-log every print. Add a one-line post-processing script in Bambu Studio, OrcaSlicer, or PrusaSlicer and each slice logs itself to your shelf — filament, colors, print time, layer height, infill, and printer, no typing.

1 Download your script

This copy has your API key already baked in — keep it private (anyone with it can post to your account).

⬇ Download printshelf_postprocess.py

Save it somewhere permanent (e.g. your Documents folder). If you re-generate your API key later, download it again.

2 Add it in your slicer

Bambu Studio / OrcaSlicer: turn on the Advanced toggle at the top of the Print Settings panel — Post-processing Scripts is hidden until you do — then open the Others tab → Post-processing Scripts. PrusaSlicer: switch to Expert mode → Print Settings → Output options → Post-processing scripts. Paste the line for your OS, editing the path to where you saved the file:

macOS / Linux /usr/bin/python3 "/Users/you/Documents/printshelf_postprocess.py"
Windows "C:\\Path\\To\\python.exe" "C:\\Users\\you\\Documents\\printshelf_postprocess.py"

Needs Python 3.8+ (no extra packages). On Windows, use the full path to python.exe.

Heads-up: when you slice, Bambu Studio shows a “Security Warning” that a post-processing script will run automatically (it may reappear each time you slice). That's expected — this script only reads the sliced G-code and posts the details to PrintShelf. Click Execute to continue. Because it runs during slicing, any action that slices — Export or Print plate — triggers it.

3 Slice something

Hit Slice, then Export or Print plate. A new print appears on your shelf within a couple seconds — with the filament matched to your library (or created if it's new). The script never changes your G-code and never blocks slicing.

Options & details
  • Public vs private: auto-logged prints are public by default. To keep them private or send them to your queue instead, open the script and set IS_PUBLIC = False or QUEUED = True near the top.
  • When it fires: at slice time (your intent to print), not on actual success — so re-slicing the same model logs it again. Delete duplicates from your shelf if needed.

Manage or rotate this key on the Account page.
