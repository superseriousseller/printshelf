📥 Auto-log your prints

Two ways to log prints without typing — from your slicer, or straight from your phone. Use either or both; prints land on your shelf automatically with title, filament, and image.

👁 Visibility

Applies to both methods below. Off = prints stay private so you can review & publish first. It's an account setting — flip it anytime, no re-download needed.

🖥 From your slicer

Add a one-line post-processing script in Bambu Studio, OrcaSlicer, or PrusaSlicer and each slice logs itself — filament, colors, print time, layer height, infill, and printer.

  1. Download your script. This copy has your API key baked in — keep it private (anyone with it can post to your account).
    ⬇ Download printshelf_postprocess.py
    Save it somewhere permanent (e.g. Documents). If you rotate your API key later, download it again.
  2. Add it in your slicer. Bambu Studio / OrcaSlicer: turn on the Advanced toggle at the top of the Print Settings panel — Post-processing Scripts is hidden until you do — then Others tab → Post-processing Scripts. PrusaSlicer: Expert mode → Print Settings → Output options → Post-processing scripts. Paste the line for your OS, editing the path to where you saved the file:
    macOS / Linux /usr/bin/python3 "/Users/you/Documents/printshelf_postprocess.py"
    Windows "C:\\Path\\To\\python.exe" "C:\\Users\\you\\Documents\\printshelf_postprocess.py"
    Needs Python 3.8+ (no extra packages). On Windows, use the full path to python.exe.
  3. Slice something. Hit Slice, then Export or Print plate. A new print appears on your shelf within a couple seconds — filament matched to your library (or created if new). The script never changes your G-code and never blocks slicing.

Heads-up: when you slice, Bambu Studio shows a “Security Warning” that a post-processing script will run automatically (it may reappear each time you slice). That's expected — this script only reads the sliced G-code and posts the details to PrintShelf. Click Execute to continue. Because it runs during slicing, any action that slices — Export or Print plate — triggers it.

📱 From your phone

Install a shortcut once, then share any model link from your phone — it lands on your shelf with the title, designer, cover image, and link. No key to paste, no computer.

  1. Install the shortcut — on your iPhone / iPad, tap Add the PrintShelf shortcut (or scan the code) → Add Shortcut.
  2. Sign in once to printshelf.app in your phone's Safari — the shortcut uses that session, so there's no API key to paste.
  3. Use it: open any model (MakerWorld, Printables, Thingiverse, Cults3D) → Share → Add to PrintShelf. Done.

Visibility follows your setting above. Re-sharing the same model won't duplicate it — even via a different link to the same model.

Options & details
  • Public vs private is the Visibility toggle at the top — an account setting that governs both the slicer and phone methods (no script editing needed).
  • Log as queued / a different status (slicer only): the script logs prints as printed by default. To send them to your queue instead, open the script and set QUEUED = True near the top; STATUS can be printed, printing, failed, or partial.
  • When the slicer script fires: at slice time (your intent to print), not on actual success — so re-slicing the same model logs it again; delete duplicates from your shelf if needed. (Phone shares dedupe automatically.)

Manage or rotate this key on the Account page.
